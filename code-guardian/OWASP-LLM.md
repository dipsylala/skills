# OWASP Top 10 for LLM Applications 2025 - Detection Guide

Use this guide only when an edit touches LLM, GenAI, RAG, embeddings, vector search, model calls, prompts, tool/function calling, autonomous agents, model training, fine-tuning, AI memory, or AI cost controls.

For each risk: signals to detect in generated code, and the fix principle to apply.

Source: <https://genai.owasp.org/resource/owasp-top-10-for-llm-applications-2025/>

| # | Risk | Detect | Fix principle |
| --- | ------ | -------- | --------------- |
| LLM01 | Prompt Injection | Untrusted user, document, webpage, email, or tool output is inserted into a prompt as instructions; retrieved content can override system/developer intent; model output controls tool arguments without policy checks | Treat all user, retrieval, and model content as untrusted data; separate instructions from data; enforce authorization and action policy outside the model; require approval for high-impact actions |
| LLM02 | Sensitive Information Disclosure | Secrets, credentials, PII, source code, tenant data, or confidential documents sent to the model unnecessarily; prompts/responses logged without redaction; training or fine-tuning uses sensitive data without controls | Minimize data sent to models; redact or tokenize sensitive values; keep secrets out of prompts; restrict retention and logging; apply data classification and tenant isolation |
| LLM03 | Supply Chain | Model, provider, package, plugin, dataset, MCP server, or prompt template added from an untrusted source; model/version not pinned; unsafe model artifact loading; no provenance for training or RAG data | Pin model and dependency versions; use trusted registries/providers; verify checksums/signatures where available; review licenses and provenance; run dependency, model, and dataset scanning where available |
| LLM04 | Data and Model Poisoning | Untrusted or user-writable data enters training, fine-tuning, evaluation, memory, or vector indexes; no provenance, review, or moderation; retrieved content can persist attacker instructions | Curate and validate datasets; separate write paths from retrieval paths; track provenance; review data before indexing or training; detect and remove poisoned content |
| LLM05 | Improper Output Handling | Model output is passed directly into SQL, shell, HTML, code execution, file paths, APIs, JSON parsers, or privileged workflows; generated code is executed without review | Treat model output as untrusted; validate against schemas; encode for output context; parameterize downstream calls; sandbox or block generated code execution |
| LLM06 | Excessive Agency | Agent has broad file, network, credential, purchase, email, deployment, or deletion permissions; tool calls auto-approve; model can chain actions without limits or checkpoints | Apply least privilege to tools and credentials; allowlist actions and targets; require human approval for high-impact operations; isolate execution; enforce quotas, timeouts, and audit logs |
| LLM07 | System Prompt Leakage | System prompts contain secrets, policy internals, credentials, private URLs, or sensitive business logic; code relies on hidden prompts as a security boundary; prompts are returned or logged | Assume system prompts can leak; never place secrets in prompts; enforce security server-side; minimize sensitive prompt content; redact prompts from logs where appropriate |
| LLM08 | Vector and Embedding Weaknesses | Retrieval ignores user, tenant, or document ACLs; metadata filters are missing or client-controlled; vector store allows cross-tenant reads; untrusted documents are retrieved as authoritative context | Enforce ACLs at retrieval time on the server; filter by tenant and document permissions; protect vector indexes; sanitize retrieved content; implement retention and deletion controls |
| LLM09 | Misinformation | Model output is used for legal, financial, medical, security, or operational decisions without grounding, confidence handling, or review; citations are missing or unverifiable; hallucinated plans are executed | Require source grounding and verification; surface uncertainty; use deterministic checks for factual claims where possible; require human review for consequential decisions |
| LLM10 | Unbounded Consumption | No token, cost, rate, tool-call, recursion, retry, or context-size limits; user can trigger expensive model/tool workflows; background agents run without cancellation | Enforce token, cost, rate, time, loop, and queue limits; add cancellation and backoff; bound retries and agent steps; monitor spend and abnormal usage |
